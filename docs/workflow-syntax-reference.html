📐 Workflow 脚本语法参考

它不是一门新语言——它就是 JavaScript + 几个注入的全局 API + 几条限制。本页是完整的语法 / API / 约束速查。

Plain JavaScript · Injected Globals · No external official URL
本质 骨架 meta 块 全局 API 语言约束 并发上限 缓存/恢复 完整示例 速查卡

一、先搞清本质

你理解得对：Workflow 多出来的就是一层中间脚本，让执行更确定（循环/分支/扇出由代码说了算，而不是模型即兴）。但这层脚本：

✅ 是标准 JavaScript

const/let、await、if/while/for、Array.map/filter/flat、JSON、Math、模板字符串、箭头函数……全都能用。

➕ 注入了 8 个全局

函数 agent parallel pipeline phase log workflow；对象 args budget。这是唯一"新"的部分。

➖ 加了几条限制

不能写 TypeScript 类型；禁用 Date.now()/Math.random()/无参 new Date()；没有文件系统/Node API。

为什么没给你外部 URL？ 因为这套 DSL 是 Workflow 工具内部定义的（在工具说明里），不是某个公开框架，没有 MDN 那样的官网。本页就是把它的完整定义整理出来供你学习。JS 本身的语法可参考 developer.mozilla.org 的 JavaScript 文档。

二、脚本骨架（必须长这样）

export const meta = { /* 见第三节，必须是第一句、纯字面量 */ }
// ↓ 主体从这里开始，运行在 async 上下文里，可直接 await
phase('Stage A')
const x = await agent('做某事', { schema: SCHEMA })
log(`拿到 ${x.count} 条`)
return { result: x }   // 顶层 return 的值会作为整个 workflow 的返回结果
两条硬规则： ① 文件第一句必须是 export const meta = {...}。② meta 必须是纯字面量——里面不能有变量、函数调用、展开运算符 ... 或模板插值。

三、meta 块字段

字段必填说明
name必填工作流名
description必填一行说明，显示在权限弹窗
phases可选数组，每个 phase() 调用对应一条 {title, detail}；title 必须和脚本里 phase('...') 完全一致
whenToUse可选显示在工作流列表里的使用场景说明
model可选某阶段用特定模型时标注
export const meta = {
  name: 'review-changes',
  description: '分维度审查改动并逐项验证',
  phases: [
    { title: 'Review', detail: '按维度并行审查' },
    { title: 'Verify', detail: '对每个发现对抗式验证' },
  ],
}

四、注入的全局 API（核心）

4.1 agent() — 启动一个子代理

agent(prompt: string, opts?: object) → Promise<string | object | null>

返回值规则：

情况返回
不传 schema子代理的最终文本（string）
传了 schema校验通过的对象（object），无需手动解析
用户中途跳过该代理null（用 .filter(Boolean) 过滤）

opts 字段：

字段类型说明
labelstring进度显示用的标签
phasestring显式归入某进度分组（在 pipeline/parallel 内部用，避免和全局 phase() 抢状态）
schemaJSON Schema强制子代理返回符合该 schema 的结构化对象
modelstring覆盖模型档位。默认不传，继承主循环模型（通常正确）
isolation'worktree'在独立 git worktree 里跑。很贵，仅当多代理并行写会冲突时用
agentTypestring用自定义子代理类型（如 'Explore'、'oh-my-claudecode:build-fixer'）
const txt  = await agent('总结这个模块')                          // 返回 string
const data = await agent('找出所有 bug', { schema: BUGS })          // 返回 object
const r    = await agent('迁移这个文件', { isolation: 'worktree',
                                            label: 'migrate:foo' })

4.2 pipeline() — 流水线（默认首选）

pipeline(items: any[], stage1, stage2, …) → Promise<any[]>

每个 item 独立穿过所有阶段，阶段间无屏障——item A 可在阶段3，item B 还在阶段1。墙钟 = 最慢的单条链路。

每个 stage 回调签名：(prevResult, originalItem, index) => … —— 后面的 stage 也能拿到最初的 item 和下标。某 stage 抛错 → 该 item 掉成 null 并跳过其余阶段。

const out = await pipeline(files,
  file               => agent(`审查 ${file}`, { schema: REVIEW }),
  (review, file, i)  => agent(`验证 ${file} 的发现`, { schema: VERDICT })
)

4.3 parallel() — 并发屏障

parallel(thunks: Array<() => Promise<any>>) → Promise<any[]>

是屏障：等全部完成才返回。注意参数是函数数组（thunk），不是 Promise 数组。某个 thunk 抛错 → 该位置变 null，调用本身永不 reject，所以用前先 .filter(Boolean)。

const res = await parallel(items.map(it => () =>   // ← 注意 () => 包一层
  agent(`处理 ${it}`, { schema: S })))
const ok = res.filter(Boolean)                       // 滤掉失败的 null
选哪个？ 默认用 pipeline()。只有当"阶段 N 需要阶段 N-1 的全部结果"（先去重再验证、统计总数后提前退出、需要互相对比）才用 parallel() 屏障。

4.4 phase() / log()

phase(title: string): void  ·  log(message: string): void

phase 开启新阶段，之后的 agent() 归到该分组；log 向用户输出一行进度叙述。

4.5 workflow() — 内联调用另一个工作流

workflow(nameOrRef: string | {scriptPath}, args?: any) → Promise<any>

把已保存的工作流当子步骤跑，只能嵌套一层（子工作流里再调 workflow() 会抛错）。

4.6 args — 入参对象

调用 Workflow 工具时传入的 args 原样出现在这里。传数组/对象要传真正的 JSON 值，别传 JSON 字符串（否则 args.map 会报错）。

// 调用方： Workflow({ script, args: ['a.cpp','b.cpp'] })
await pipeline(args, f => agent(`处理 ${f}`))

4.7 budget — token 预算

budget.total: number|null  ·  budget.spent(): number  ·  budget.remaining(): number

用户用 "+500k" 之类设的本轮目标。total 为 null 表示没设（此时 remaining() 是 Infinity）。可做动态深度循环：

while (budget.total && budget.remaining() > 50_000) {
  const r = await agent('继续找问题', { schema: BUGS })
  bugs.push(...r.bugs)
}

五、语言约束（容易踩坑）

规则原因 / 替代
✗不能写 TypeScript：类型标注 : string[]、interface、泛型 <T>解析会失败。就写纯 JS
✗Date.now() / Math.random() / 无参 new Date()会抛错（破坏 resume）。时间戳通过 args 传入；要随机就靠 item 下标变 prompt
✗文件系统 / Node.js API（fs、require 等）不可用。读写文件让 agent() 去做
✓标准内置：JSON Math Array Object Set Map 等正常用
✓顶层直接 await（脚本跑在 async 上下文）不用包 async function
✓自定义辅助函数（如 chunk()）可以在脚本里 function chunk(){}

六、并发与上限

限制值
同时运行的代理数min(16, CPU核数 - 2)，超出的排队
单个 workflow 生命周期代理总数1000（防失控的后备上限）
嵌套 workflow() 层数仅 1 层

所以你尽管给 parallel()/pipeline() 传 100 个 item，它们都会完成，只是任意时刻只有约 10 个在跑。

七、缓存与恢复（Resume）

每次调用 Workflow 都会把脚本落盘，返回一个 scriptPath 和 runId。这正是"用户可以改脚本来更确定执行"的关键机制：

操作做法
迭代脚本编辑那个 scriptPath 文件，再用 Workflow({ scriptPath }) 重跑
断点续跑Workflow({ scriptPath, resumeFromRunId })：未改动的 agent() 调用秒回缓存，只重跑第一个改动点及之后
完全命中同脚本 + 同 args → 100% 缓存命中
这也是为什么禁用 Date.now()/Math.random()：它们会让"同脚本同输入"产生不同结果，破坏缓存命中和可复现性。

八、覆盖几乎所有语法点的完整示例

把上面的语法点串起来——这是一个"找 bug → 去重 → 多镜头对抗验证 → 循环到枯竭"的真实结构：

export const meta = {
  name: 'exhaustive-bug-hunt',
  description: '穷尽式找 bug 并多镜头验证',
  phases: [{ title: 'Find' }, { title: 'Verify' }],
}

// 自定义辅助函数 —— 纯 JS，允许
function key(b) { return b.file + ':' + b.line }

const FINDERS = [
  { prompt: '按数据流找空指针/越界' },
  { prompt: '按并发找数据竞争' },
  { prompt: '按资源生命周期找泄漏' },
]
const seen = new Set(), confirmed = []
let dry = 0

while (dry < 2) {                                   // 循环到枯竭
  phase('Find')
  const found = (await parallel(FINDERS.map(f => () =>   // parallel 屏障 + thunk
    agent(f.prompt, { phase: 'Find', schema: BUGS }))))
    .filter(Boolean).flatMap(r => r.bugs)        // 滤 null + 摊平

  const fresh = found.filter(b => !seen.has(key(b)))  // 对 seen 去重
  if (!fresh.length) { dry++; continue }
  dry = 0; fresh.forEach(b => seen.add(key(b)))
  log(`本轮新发现 ${fresh.length} 个`)               // 进度叙述

  phase('Verify')
  const judged = await parallel(fresh.map(b => () =>
    parallel(['correctness', 'security', 'repro'].map(lens => () =>  // 多镜头
      agent(`用 ${lens} 视角判断 "${b.desc}" 是否真实`,
            { phase: 'Verify', schema: VERDICT })))
      .then(vs => ({ b, real: vs.filter(Boolean)
                                .filter(v => v.real).length >= 2 }))))  // 多数票

  confirmed.push(...judged.filter(v => v.real).map(v => v.b))
}

return { confirmed, total: confirmed.length }      // 顶层返回

九、一页速查卡

// ── 结构 ──────────────────────────────
export const meta = { name, description, phases:[{title,detail}] }  // 第一句·纯字面量

// ── 注入全局（唯一"新"语法）─────────────
await agent(prompt, { label, phase, schema, model, isolation, agentType })
await pipeline(items, stageA, stageB, ...)   // 无屏障；stage(prev, item, i)
await parallel(items.map(x => () => ...))    // 屏障；thunk 数组；失败→null
phase(title)   log(msg)   await workflow(name, args)   // 嵌套限 1 层
args                                       // 入参（真 JSON，非字符串）
budget.total / budget.spent() / budget.remaining()

✗ Date.now() / Math.random() / new Date()
✗ fs / require / Node API
